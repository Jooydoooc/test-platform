# Ranking and Motivation

The ranking system motivates students — it must never embarrass weak students. This is a hard constraint on every feature below, not a soft preference.

## Core privacy/visibility rules (restated explicitly here, not just implied elsewhere)

- All ranking and leaderboards are scoped to a student's **own group only** — never cross-group, since curriculum levels differ (a Beginner is never ranked against IELTS Graduation).
- Students appear by **first name + last initial only** (e.g. "Aziz K."), never full name.
- **Public leaderboards show only the top 5** per category. Every student can always see their **own exact rank and score privately** ("You're #9 in Vocabulary Champion this week"), but the full bottom-of-list ranking of classmates is never shown publicly. This is what actually prevents embarrassment — having multiple categories helps, but a full public list still exposes who's last on each one.

## Ranking categories

Multiple categories, not one dominant leaderboard — so different kinds of effort are visible, not just raw top scorers:

- **Top score** — highest average score, per skill or overall, within the group.
- **Most improved** — largest positive change using the exact monthly comparison metric defined in `PROGRESS_TRACKING.md` (this month's average vs. last month's, per skill). Not a separately invented metric.
- **Most active** — total completed tasks across all categories (Practice, Homework, Tests, Extra Practice), using the same count defined in `PROGRESS_TRACKING.md`.
- **Longest streak** — scheduled-session-day streak, per `STUDENT_EXPERIENCE.md`.
- **Vocabulary champion** — highest vocabulary mastery/accuracy, using the state definitions in `VOCABULARY_SYSTEM.md`.

## Points

Points have defined base values per action (exact numbers are a tunable config value in `CLAUDE_RULES.md`, not hardcoded):
- Completing a task
- Taking a test
- A skill's trend indicator moving positive (i.e. genuine improvement, per the trend definition in `PLATFORM_PURPOSE.md` — not awarded per attempt regardless of result)
- Reviewing a mistake
- Learning a new vocabulary item (reaching "Learned" or "Mastered" state, per `VOCABULARY_SYSTEM.md`)
- Keeping a streak alive
- Finishing a textbook unit

**Anti-gaming rule:** points are awarded once per unique, meaningful action. Repeating an already-reviewed mistake or re-practising an already-mastered vocabulary item does not re-award points — this prevents point-farming through trivial repetition.

## Badges

Badges follow a general unlock pattern (milestone count within a skill or activity type), so new badges can be added later without custom one-off logic each time:

- Grammar Starter / Vocabulary Builder / Reading Climber / Listening Focus / **Writing Voice** / **Speaking Confidence** — first-milestone badges per skill area (all six, not just four — consistent with all six skills now being in scope).
- 7-Day Streak
- Top 3 This Week
- Most Improved Student
- Unit Master (completing all tasks in a textbook unit)

## Weekly challenges

**System-generated by default**, pulled from a rotating template pool (e.g. "complete 10 vocabulary reviews this week," "finish 2 reading tasks this week") rather than authored fresh by the teacher every week — manually writing new challenges across 7 groups weekly is exactly the kind of repeated effort this platform should remove. A teacher can override or customize a challenge for a specific group if they want to, but it's not required.
